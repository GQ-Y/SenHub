# 雷达与球机坐标系对齐设计

## 1. 目标

将**雷达坐标系**下的点云位置，变换到**球机（摄像头）坐标系**，从而由目标在雷达中的 (x, y, z) 计算球机的 pan/tilt 角度，实现 PTZ 联动与抓拍。  
对齐方式：**以球机为参考原点**，通过「平移 + 旋转 + 缩放」把雷达坐标变换到球机坐标系。

---

## 2. 坐标系定义（必须统一约定）

### 2.1 雷达坐标系（Livox MID-360）

- **原点**：雷达几何中心（或安装参考点）。
- **轴向**（右手系，单位：米；SDK 原始为 mm/cm，转换后使用米）：
  - **X**：设备右侧为正  
  - **Y**：设备正前方为正  
  - **Z**：设备正上方为正  
- 点云中的 `Point(x, y, z)` 即在该系下的坐标。

> 若实际设备/固件与上述不一致，只需在标定得到的旋转中多一个固定轴旋转即可（见下文标定）。

### 2.2 球机（摄像头）坐标系（本系统约定）

- **原点**：球机云台旋转中心（或镜头光学中心，取决于你希望“对准”的参考点）。
- **轴向**（右手系）：
  - **X**：球机“右侧”方向为正（水平面内）
  - **Y**：球机“正前方”方向为正（水平面内，镜头正对方向）
  - **Z**：向上为正  

- **与 PTZ 角度的关系**（与当前 `CoordinateTransform.calculatePTZAngles` 一致）：
  - **Pan（水平角）**：从 **X 轴** 绕 **Z 轴** 转向 **Y 轴** 为正方向；  
    `pan = atan2(y, x)`，再映射到 [0°, 360°)。
  - **Tilt（俯仰角）**：从水平面（X-Y）向 **Z 轴** 仰为正；  
    `tilt = atan2(z, sqrt(x² + y²))`。

因此：**球机坐标系下** 任意一点 (x, y, z)，用上述公式即可得到 (pan, tilt)。

---

## 3. 统一到“球机参考系”的变换

思路：**先把雷达坐标变换到“球机坐标系”，再按上文公式算 pan/tilt。**

变换采用：**缩放 → 旋转 → 平移**（与当前 `CoordinateTransform` 一致）。

- **缩放**：  
  `(x', y', z') = scale * (x, y, z)`  
  一般 scale=1；若雷达与球机单位不一致或用错单位时可调。

- **旋转**：欧拉角（度），顺序 **Z → Y → X**（绕 Z、再绕 Y、再绕 X）。  
  将“雷达系下的向量”变到“球机系下的向量”。

- **平移**：  
  `(x_cam, y_cam, z_cam) = (x', y', z') + (tx, ty, tz)`  
  表示**雷达原点在球机坐标系下的位置**（米）。

得到球机系下点 `(x_cam, y_cam, z_cam)` 后：

- `pan = atan2(y_cam, x_cam)`，并转到 [0, 360)  
- `tilt = atan2(z_cam, sqrt(x_cam² + y_cam²))`

即可驱动 PTZ。

---

## 4. 标定参数从哪里来（如何“同频”）

“同频”即：**标定出一组 (平移, 旋转, 缩放)**，使雷达里的点变换到球机系后，计算出的 pan/tilt 与实际“球机对准该点”时的 pan/tilt 一致。

### 4.1 准备

- 雷达、球机**固定安装**后不再移动。
- 明确：雷达坐标系轴向、球机坐标系轴向（如上），以及本系统里 pan/tilt 的约定（如上）。

### 4.2 推荐做法：多点标定

1. **在场景中选 2～4 个特征点**（如墙角、标杆、明显物体），要求：
   - 既能在点云里稳定看到（能得到雷达坐标），  
   - 又能让球机转向该点并在画面上精对准（能读到 pan/tilt）。

2. **对每个点 i**：
   - 从点云取该点在**雷达系**下的坐标 `(x_i, y_i, z_i)`（可多帧平均）。
   - 球机对准该点，从设备或平台读取当前 **pan_i, tilt_i**。

3. **由 (pan_i, tilt_i) 反算球机系下的方向向量**（单位长度即可）：  
   - `horizontal = cos(tilt_i * π/180)`  
   - `x_cam_i = cos(pan_i * π/180) * horizontal`  
   - `y_cam_i = sin(pan_i * π/180) * horizontal`  
   - `z_cam_i = sin(tilt_i * π/180)`  

   若希望“距离”一致，可用一个已知距离 d，得到球机系下点：  
   `(d*x_cam_i, d*y_cam_i, d*z_cam_i)`。

4. **建立方程**：  
   设变换为 旋转 R + 平移 t，有：  
   `[x_cam_i; y_cam_i; z_cam_i] = R * [x_i; y_i; z_i] + t`。  
   用 2 个以上点可解 R、t（可用 PnP、SVD 等）；若只做平移则 1 个点即可。

5. **把 R 转为欧拉角 (Z→Y→X)**，与 t、scale 一起写入防区的 **坐标系转换参数**（见下）。

### 4.3 快速粗对齐（只做平移）

- 若雷达与球机**同高、同向**安装，可近似认为只有平移：
  - 测量雷达原点在“球机坐标系”下的 (tx, ty, tz)（单位：米），  
  - 设 rotation=0、scale=1，仅设置 translation。
- 用一个已知点做一次“对准”，微调 tx, ty, tz 直到 pan/tilt 与计算值接近。

### 4.4 参数写入与使用

- 防区表/配置中有 **coordinateTransform**（JSON），例如：

```json
{
  "translation": { "x": 0.0, "y": 0.0, "z": 0.0 },
  "rotation": { "x": 0, "y": 0, "z": 0 },
  "scale": 1.0
}
```

- 标定得到的 **平移 (tx, ty, tz)** 填入 `translation`。  
- 标定得到的 **欧拉角 (Z→Y→X，度)** 填入 `rotation`（若用弧度需先转成度）。  
- 后端 `CoordinateTransform` 会按「缩放 → 旋转(Z→Y→X) → 平移」施放到雷达点，再算 pan/tilt，即实现“同频”。

---

## 5. 安装与约定小结（便于现场对齐）

| 项目       | 说明 |
|------------|------|
| 雷达原点   | 取雷达几何中心或安装标记点，与后续测量一致 |
| 雷达轴向   | X 右、Y 前、Z 上（右手系），单位米 |
| 球机原点   | 云台旋转中心（或约定为某固定点） |
| 球机轴向   | X 右、Y 前、Z 上；pan=atan2(y,x)，tilt=俯仰 |
| 变换顺序   | 缩放 → 旋转(Z→Y→X) → 平移 |
| 标定输入   | 至少 1 点（仅平移）或 2～4 点（旋转+平移） |
| 参数存放   | 防区 `coordinateTransform` JSON |

---

## 6. 与本项目代码的对应关系

- **DefenseZone.coordinateTransform**（JSON）  
  → 解析为 `translation`、`rotation`（度）、`scale`。  
- **CoordinateTransform.transformRadarToCamera(Point)**  
  → 雷达点 → 球机系点。  
- **CoordinateTransform.calculatePTZAngles(Point)**  
  → 球机系点 → (pan, tilt)。  

只要标定出的 (translation, rotation, scale) 与上述约定一致，雷达与球机的坐标系就实现了“同频”对齐。

## 7. 可选：标定辅助界面思路

后续可在“防区配置”或“雷达/球机配置”中增加“坐标系标定”入口：

1. **采集**：用户选择当前防区，在场景中选 1～4 个点；对每点触发“记录当前点云中心”与“记录当前 PTZ 角度”。  
2. **计算**：后端用多点解算 R、t，并转成欧拉角 + 平移。  
3. **回写**：将结果写入该防区的 `coordinateTransform`，并支持微调（表单输入 translation/rotation 的数值）。  

这样“同频”设计就通过标定流程在界面上闭环完成。
